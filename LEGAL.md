# TUTODECODE - Politique legale et licences

## 1) Repartition des licences

### WebApp (React / Vite)
- Licence: GNU AGPL v3
- Portee: code web applicatif du depot.
- Effet: si une version modifiee est exposee via un serveur, le code source modifie doit etre rendu disponible.

### Logiciel desktop CourseBuilderExe (C#)
- Licence: GNU GPL v3
- Portee: tools/CourseBuilderExe/

### Contenu pedagogique (cours JSON, images, medias)

- Portee: contenus de cours et ressources pedagogiques.
- Conditions:
  - BY: attribution obligatoire a Association TUTODECODE

  - SA: partage sous la meme licence

### Pont IA / documentation d'integration
- Licence: MIT
- Portee: elements techniques de connexion/integration IA et leur documentation.

## 2) Transparence et securite
- Le code source de l'application principale est public.
- Les binaires officiels doivent etre telecharges uniquement depuis les canaux officiels.

## 3) Vie privee
- Principe privacy-first.
- Aucune collecte de donnees personnelles par defaut.
- Pas de tracking publicitaire integre.
- Les donnees utilisateur restent locales tant qu'aucun service externe n'est explicitement active.

### Garantie de souverainete et absence de telemetrie
- Engagement: aucune telemetrie tierce, aucun traqueur publicitaire et aucun SDK de suivi ne sera ajoute au code officiel.
- Toute proposition de contribution qui introduit un mecanisme de suivi tiers doit etre refusee.
- Cette garantie s'applique au code publie dans ce depot officiel.

## 4) Marque et identite visuelle (Trademark)
- La licence AGPL s'applique au code, mais ne couvre pas le nom "TutoDeCode" ni le logo associe.
- L'utilisation du nom, du logo ou de la charte graphique est reservee a l'Association TUTODECODE.
- Toute version modifiee (fork) doit etre renommee de maniere explicite et ne doit pas creer de confusion avec le projet officiel.

## 5) Contributions et CLA
- Toute contribution est soumise au CLA du projet.
- Le CLA est disponible dans le fichier CLA.md et doit etre accepte par tout contributeur.

## 6) Absence de garantie
Le projet est fourni "en l'etat" ("as is"), sans garantie explicite ou implicite.
